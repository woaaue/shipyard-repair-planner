Version 2Разные рабочие поверхности

Прототипы без шаблонной hero-структуры

Каждый экран собран под реальную роль: клиент видит прозрачный статус ремонта, производство работает по цепочке рабочий -> мастер -> оператор дока -> диспетчер. Админ управляет ролями и привязками.

КлиентСтатус ремонта, согласования, смета, документы и подтвержденные события. РабочийОдин текущий наряд, шаги, фотофиксация и прямой мастер. МастерРабочие, посты, статусы этапов и передача оператору дока. ОператорОдин док, несколько мастеров, приемка этапов и связь с диспетчером. ДиспетчерНесколько доков, операторы, ресурсные конфликты и общий график. АдминПользователи, роли, матрица доступа и аудит подчиненности.

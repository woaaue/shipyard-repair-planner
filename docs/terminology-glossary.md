# Глоссарий терминов интерфейса

Этот словарь фиксирует единые формулировки для фронтенда и документации.

## Основные сущности
- `Заявка` — запрос клиента на проведение ремонта (`repair request`).
- `Ремонт` — подтвержденная и выполняемая работа по заявке (`repair`).
- `Задача` — отдельная работа внутри ремонта (`work item`).
- `Приемка` — подтверждение клиентом завершенного ремонта.

## Роли
- `Администратор`
- `Диспетчер`
- `Оператор дока`
- `Мастер участка`
- `Рабочий`
- `Клиент`

## Обязательные правила UI
- Использовать `Клиент`, а не `Владелец судна`.
- Использовать `Оператор ремонта`/`Оператор дока`, а не `Менеджер`.
- Использовать `Заявка`, а не `Request`, `Repair Request`, `тикет`.
- Технические enum-значения (`IN_PROGRESS`, `NOT_SUBMITTED` и т.п.) не показывать напрямую пользователю.
